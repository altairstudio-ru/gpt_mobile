On branch fix/safer-image-upload-pipeline, user reported retry still ANR-like and send paused before chat processing. Root cause investigation showed provider request builders in ChatRepositoryImpl were still building full request/history/attachment structures before emitting ApiState.Loading, and completeChat()/retryChat() call chatRepository.completeChat from viewModelScope.launch on Main. This meant expensive history/image preparation blocked the main path during send and retry. Fix: add internal helper streamPreparedApiState() that emits ApiState.Loading immediately, then runs request preparation in withContext(Dispatchers.Default), then emitAll()s the provider stream flow. Applied to OpenAI Responses, OpenAI Chat Completions, Anthropic, and Google paths. Added regression test in ChatRepositoryImplTest asserting Loading is emitted before expensive preparation finishes.
