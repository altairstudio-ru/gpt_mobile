Issue #253 root cause investigation: chat requests re-encode every historical image attachment on each send. `FileUtils.readAndEncodeFile()` previously loaded the full file with `readBytes()` and then base64-encoded it, creating large transient allocations. This became especially bad with multiple high-resolution images and could make long chats crash with OOM. Fix approach: add sampled image encoding in `FileUtils` (2048px max edge, JPEG/PNG/WEBP output as appropriate, fallback to original stream for GIF/SVG), stream base64 for fallback encoding, and update OpenAI/Anthropic/Google request builders to use the normalized encoded image mime type returned by `readAndEncodeImageForUpload()`.
